Visão geral

Onde os dados ficam e como o fluxo checkout → PIX → webhook se conecta à Quantum.

Onde os dados do painel são salvos?

Com o Node.js (Railway ou npm start na pasta server/):
· server/data/config.json — chaves do gateway, token Utmify, GA/Ads, URL pública do site, limite de ingressos por pedido, hash da sua senha de admin.
· server/data/orders.json — lista de todos os pedidos (cliente, PIX, status, UTM, IDs).

Na Railway, sem volume montado em /app/server/data, esses arquivos podem ser perdidos a cada redeploy. Para produção, adicione um volume ou backup externo.

Na HostGator (PHP), o equivalente fica na pasta data/ na raiz do site (ver api/index.php / lib/Store.php).

Checklist rápido

1) Gateway PIX — chaves Quantum + unidade de valor (centavos/reais). 2) URL pública — igual à URL do app na Railway (webhook). 3) Utmify e Google conforme uso. 4) Salvar tudo no topo. 5) Testar checkout em aba anônima.

Gateway PIX (Quantum)

Autenticação Basic com chave pública e secreta. Se o PIX retornar erro de valor, experimente trocar a unidade abaixo conforme o PDF/manual da Quantum.

Utmify e URL pública

          
Webhook do PIX e postback do gateway Quantum usam esta base (mesma URL HTTPS que o cliente abre, ex. Railway). Sem isso a Quantum recusa com “URL de postback inválida”.

        

        

          
Credenciais e URL

          
Token de API da Utmify e a URL base do site para o webhook do PIX (ex.: https://seu-app.up.railway.app).
